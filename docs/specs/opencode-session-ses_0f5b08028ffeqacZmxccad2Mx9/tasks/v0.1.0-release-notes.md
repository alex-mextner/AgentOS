## v0.1.0 — initial release (Phase 0–1: bootstrap + mobile scaffold + GitHub client)

First tagged release of **HyperGit** — an open-source GitHub replacement with a local-first SwiftUI mobile app.

> Early / pre-alpha. The mobile MVP scaffold is in place; the platform backend (server/) is not yet implemented (Phase 3). See `docs/SPEC.md`.

### What's included
- **Spec & conventions** — master spec (`docs/SPEC.md`: platform §1, mobile §2, non-goals §3, alternatives research §5, roadmap §6), `AGENTS.md`, rig-managed guardrails (secret-scan, CodeQL, dependency-review, PR-checklist, review-threads, leftover-grep).
- **Mobile core (`HyperGitCore`)** — models, injectable `HTTPTransport`, Keychain `TokenStore`, `GitHubClient` (REST v3) + `LinearClient` (GraphQL) behind `RepositorySource` / `TicketSource` protocols, `CacheStore`, `@Observable AppStore` (local-first cache fallback). 16 unit tests.
- **iOS app** — `@main` SwiftUI app, tabs (Repos / Tickets / Search / Settings), repo detail (Files / Pulls / Issues / Commits), file viewer, PR detail with diff, settings for GitHub PAT + Linear key. Generated via `xcodegen` (`project.yml`).
- **CI** — `mobile` workflow: `swift test` + iOS Simulator `xcodebuild` build on `macos-latest` (green).

### Build
```sh
cd mobile
swift test                 # core (no Xcode needed)
brew install xcodegen && xcodegen generate
xcodebuild -scheme HyperGit -destination 'generic/platform=iOS Simulator' build
```

### Artifact
`HyperGit-iOS-Simulator-v0.1.0.app.zip` — an unsigned Simulator `.app` (no Apple account needed). Run it:
```sh
unzip HyperGit-iOS-Simulator-v0.1.0.app.zip
xcrun simctl boot 'iPhone 15' || true
xcrun simctl install booted HyperGit.app
xcrun simctl launch booted ai.hypergit.mobile
```

### Known limitations
- GitHub `contents` API errors on files >1 MB (Git Blobs fallback is a tracked follow-up).
- Local-first cache is in-memory (`MemoryCacheStore`); SwiftData persistence lands in #4.
- No platform backend yet (#6 decides Forgejo/Gitea fork vs custom).
